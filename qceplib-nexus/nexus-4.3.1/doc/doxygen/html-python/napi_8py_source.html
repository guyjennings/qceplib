5 # @package nxs.napi
6 # Wrapper for the NeXus shared library.
7 #
8 # Use this interface when converting code from other languages which
9 # do not support the natural view of the hierarchy.
10 #
11 # Library Location
12 # ================
13 #
14 # This wrapper needs the location of the libNeXus precompiled binary. It
15 # looks in the following places in order::
16 #
17 # @verbatim
18 # os.environ['NEXUSLIB'] - All
19 # directory containing nxs.py - All
20 # os.environ['NEXUSDIR']\\bin - Windows
21 # os.environ['LD_LIBRARY_PATH'] - Unix
22 # os.environ['DYLD_LIBRARY_PATH'] - Darwin
23 # PREFIX/lib - Unix and Darwin
24 # /usr/local/lib - Unix and Darwin
25 # /usr/lib - Unix and Darwin
26 # @endverbatim
27 #
28 # - On Windows it looks for one of libNeXus.dll or libNeXus-0.dll.
29 # - On OS X it looks for libNeXus.0.dylib
30 # - On Unix it looks for libNeXus.so.0
31 # - NEXUSDIR defaults to 'C:\\Program Files\\NeXus Data Format'.
32 # - PREFIX defaults to /usr/local, but is replaced by the value of --prefix during configure.
33 #
34 # The import will raise an OSError exception if the library wasn't found
35 # or couldn't be loaded. Note that on Windows in particular this may be
36 # because the supporting HDF5 dlls were not available in the usual places.
37 #
38 # If you are extracting the nexus library from a bundle at runtime, set
39 # os.environ['NEXUSLIB'] to the path where it is extracted before the
40 # first import of nxs.
41 #
42 # Example
43 # =======
44 # @code
45 # import nxs
46 # file = nxs.open('filename.nxs','rw')
47 # file.opengroup('entry1')
48 # file.opendata('definition')
49 # print file.getdata()
50 # file.close()
51 # @endcode
52 #
53 # See @see nxstest.py for a more complete example.
54 #
55 # Interface
56 # =========
57 #
58 # When converting code to python from other languages you do not
59 # necessarily want to redo the file handling code. The nxs
60 # provides an interface which more closely follows the
61 # NeXus application programming interface (NAPI_).
62 #
63 # This wrapper differs from NAPI in several respects::
64 #
65 # - Data values are loaded/stored directly from numpy arrays.
66 # - Return codes are turned into exceptions.
67 # - The file handle is stored in a file object
68 # - Constants are handled somewhat differently (see below)
69 # - Type checking on data/parameter storage
70 # - Adds iterators file.entries() and file.attrs()
71 # - Adds link() function to return the name of the linked to group, if any
72 # - NXmalloc/NXfree are not needed.
73 #
74 # File open modes can be constants or strings::
75 #
76 # @verbatim
77 # nxs.ACC_READ 'r'
78 # nxs.ACC_RDWR 'rw'
79 # nxs.ACC_CREATE 'w'
80 # nxs.ACC_CREATE4 'w4'
81 # nxs.ACC_CREATE5 'w5'
82 # nxs.ACC_CREATEXML 'wx'
83 # @endverbatim
84 #
85 # Dimension constants::
86 #
87 # - nxs.UNLIMITED - for the extensible data dimension
88 # - nxs.MAXRANK - for the number of possible dimensions
89 #
90 # Data types are strings corresponding to the numpy data types::
91 #
92 # 'float32' 'float64'
93 # 'int8' 'int16' 'int32' 'int64'
94 # 'uint8' 'uint16' 'uint32' 'uint64'
95 #
96 # Use 'char' for string data.
97 #
98 # You can use the numpy A.dtype attribute for the type of array A.
99 #
100 # Dimensions are lists of integers or numpy arrays. You can use the
101 # numpy A.shape attribute for the dimensions of array A.
102 #
103 # Compression codes are::
104 #
105 # 'none' 'lzw' 'rle' 'huffman'
106 #
107 # As of this writing NeXus only supports 'none' and 'lzw'.
108 #
109 # Miscellaneous constants::
110 #
111 # - nxs.MAXNAMELEN - names must be shorter than this
112 # - nxs.MAXPATHLEN - total path length must be shorter than this
113 # - nxs.H4SKIP - class names that may appear in HDF4 files but can be ignored
114 #
115 # Caveats
116 # =======
117 #
118 # @todo NOSTRIP constant is probably not handled properly,
119 # @todo Embedded nulls in strings is not supported
120 #
121 # @warning We have a memory leak. Calling open/close costs about 90k a pair.
122 # This is an eigenbug:
123 # - if I test ctypes on a simple library it does not leak
124 # - if I use the leak_test1 code in the nexus distribution it doesn't leak
125 # - if I remove the open/close call in the wrapper it doesn't leak.
126 #
127 #
